Important information

  • In case you are using 1 pin Emulation POD USB with ISO you can speed up the communication by doing the following: Select Device Manager and expand the Ports(COM & LPT) setting. Open properties for 1-pin Emulation POD USB with ISO and select the Port Settings property page. Click the Advanced... button and set Latency Timer to 1 msec.

New features

  • The cache used by the Memory window will now update itself at runtime to also reflect changes to code memory.

  • When the device is programmed, the contents of the data flash area is now retained by default. A new option, Erase data flash, has also been added. When selected, this option will clear the contents of the data flash area when the device is programmed.

Known Problems

  • The following devices are not supported:

    • ADUC812
    • ADUC814
    • ADUC816
    • ADUC824

Release history

V7.51A, March 2009

No changes.

V7.50A, June 2008

  • The stack plugin is now able to initialize the extended stack correctly.

V7.40A, February 2008

  • Sometimes breakpoints were not removed at the end of a debug session due to a problem with the CODE memory cache of the driver. This has been corrected.

V7.30B, September 2007

No changes.

V7.21A, March 2007

  • When single stepping through C code, sometimes stepping over a return statement could cause the debugger to loose track of its current position.

V7.20H, Nov 2006

  • The driver now supports both ADu and ADe devices.

  • The driver has been updated to improve stability and download speed.

V7.20D, May 2006

  • There is a new option named 'CPU core clock frequency'. Here are the frequencies for some of the boards:

    • ADUC831 11059200 (if the 11.0592MHz crystal is used)
    • ADUC832 16777216
    • ADUC834 12582912
    • ADUC836 12582912
    • ADUC841 11059200 (if the 11.0592MHz crystal is used)
    • ADUC842 16777216
    • ADUC843 16777216
    • ADUC845 12582912
    • ADUC846 12582912
    • ADUC847 12582912
    • ADUC848 12582912

  • The communication between the host and the device has been improved. This improvement will make all the debug commands execute faster.

V7.20B, Nov 2005

  • It is now possible to select among more than just 4 different COM ports.

V7.20A, Oct 2005

No changes.

V7.10A, July 2005

Initial release.
